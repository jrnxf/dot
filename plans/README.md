## Findings considered and rejected

- `curl | sh` installers (Homebrew, Claude Code, omz, nvm): standard
  distribution mechanism for these tools; pinning/vendoring adds maintenance
  for negligible risk reduction on a personal machine.
- `yolo`/`claude` aliases (`zshrc:118-119`) both resolving to
  `--dangerously-skip-permissions` via zsh alias chaining: consistent with
  `skipDangerousModePermissionPrompt: true` in claude settings — intentional
  posture, not a bug. (If `yolo` was meant to be the *only* dangerous entry
  point, it isn't — but that's the operator's call, not a fix.)
- Absolute `/Users/colby/...` paths in `config/claude/settings.json` hooks
  and statusline: single-user, single-platform repo; parameterizing buys
  nothing.
- `kj()` parsing `jobs -l` with `awk '{print $3}'` (`zshrc:236-238`):
  fragile but low-stakes and easy to notice when it fails.
- `docs/nvim.md`: generic tips, tiny, harmless — not worth deleting.
- Permissive Claude permission allowlist (`Bash(curl:*)`, `Bash(pkill:*)`,
  `defaultMode: acceptEdits`): deliberate user preference, documented
  posture; flagged once here, not a plan.
- Dotbot submodule currency: pinned submodule on a stable tool; update is
  routine maintenance, not a finding.
